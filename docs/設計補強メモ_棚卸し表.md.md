## 4. 実装参照優先順序（仮）

Claude Codeが実装時に参照すべき順序。

```
① 実装参照文書（Step 3で作成）        ← 最初に読む。全体像・制約・参照誘導
② 概要設計書（フェーズ2最終確定版 v3） ← 設計の正文
③ DATA-01                          ← 状態管理の正文（最高優先）
④ FLOW-01                          ← フェーズBステップ順序の正文（最高優先）
⑤ UI-01                            ← 画面・端末役割の正文
⑥ 各補強メモ（①に誘導された箇所のみ） ← 実装対象機能に応じて個別参照
⑦ 要件定義書（フェーズ1 Rev.4）       ← 機能要件の根拠確認時のみ
```
