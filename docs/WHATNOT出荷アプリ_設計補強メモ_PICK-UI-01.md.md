# 設計補強メモ

論点ID：PICK-UI-01
タイトル：ピッキング画面の基本操作構造
ステータス：確定済み

---

## 0. 設計思想

スキャン操作と数量確定を分離することで、
過剰ピッキング・誤カウントを構造的に防ぐ。

・スキャン ＝ 商品特定のみ
・数量入力 ＝ 数量確定

スキャン成功時点ではカウントを増やさない。

---

## 1. 全体フロー

```
一覧画面
　├─ スキャン開始 → スキャン画面
　│　　├─ JAN一致 → 数量入力画面 → 一覧画面へ戻る
　│　　├─ JAN不一致 → アラート → スキャン画面に留まる
　│　　└─ 明示操作で一覧画面へ戻る
　└─ 商品選択（スキャン不能時）→ 数量入力画面 → 一覧画面へ戻る
```

---

## 2. 画面仕様

### ① 一覧画面

| 項目 | 内容 |
|---|---|
| 役割 | 進捗確認・操作起点 |
| 表示内容 | 商品名・必要数量・スキャン済み数量・残数 |
| 必要数量の更新 | ピッキング中の注文追加に伴い随時更新される |
| 操作① | 「スキャン開始」ボタン → スキャン画面へ遷移 |
| 操作② | 商品選択 → 数量入力画面へ直接遷移（スキャン不能時） |

### ② スキャン画面

| 項目 | 内容 |
|---|---|
| 役割 | バーコード読み取り・商品特定のみ |
| 前提 | 連続スキャン前提。1回スキャンするたびに商品特定 |
| JAN一致時 | 商品を特定 → 数量入力画面へ遷移。この時点ではカウントしない |
| JAN不一致時 | アラート表示・進捗に反映しない・スキャン画面に留まる |
| 手入力切替 | スキャン画面上の切替操作 → 数量入力画面へ遷移。対象商品が特定済みの場合のみ有効。未特定の場合は一覧画面で商品選択を行う |
| 一覧へ戻る | 明示的な操作でのみ一覧画面へ戻る |

### ③ 数量入力画面

| 項目 | 内容 |
|---|---|
| 役割 | 数量確定・進捗記録 |
| 遷移元 | スキャン画面（JAN一致後）または一覧画面（商品選択後） |
| 表示内容 | 商品名・最新の必要数量・現在のスキャン済み数量・残数 |
| 入力内容 | 今回追加でピッキングした数量（追加数量。総数ではない） |
| 入力上限 | 残数まで。上限超過入力はエラーとして受け付けない |
| 確認 | 確認ダイアログ表示後に確定 |
| 確定後 | 進捗に加算・一覧画面へ戻る |

---

## 3. 数量ルール

| ルール | 内容 |
|---|---|
| 入力単位 | 追加数量（総数ではない） |
| 上限 | 残数（必要数量－スキャン済み数量） |
| 上限超過 | エラー。入力を受け付けない |
| 複数回ピッキング | 同一商品を複数回に分けてピッキング可。都度加算される |
| 必要数量の変動 | ピッキング中の注文追加で増加した場合、残数は自動再計算される |

---

## 4. リスト外商品スキャン時の挙動

- リスト外JANコードをスキャン → アラート表示・進捗に反映しない
- スキャン画面に留まり、次の操作を続ける

---

## 5. JANコード前提

- JANコードは完全にユニーク
- JAN一致 ＝ 商品特定確定
- 同一JANが複数商品に紐づくケースは考慮不要

---

## 6. 禁止事項

- スキャン時の自動カウントアップ禁止
- 数量入力画面での上限超過入力許可禁止
- リスト外JANコードの進捗反映禁止
- 商品属性としての「バーコードなし商品」を前提にすることは禁止
- 対象商品が未特定の状態でスキャン画面から数量入力画面へ遷移することは禁止
